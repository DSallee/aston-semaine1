Versionner un projet en équipe

Ordre des commandes de Versionning

  • git init se lance 1 seule fois par projet, créé le fichier .git
  • git status indique l'état
  • git add traque les modifications, les versions
  • git commit -m "message_name" valide les modifications, les versions
  • git push -u origin master envoi sur le cloud GitHub

Rappel des commandes de base

  • cd change de répertoire.
  • ls contenu du répertoire.
  • mkdir créé un répertoire.
  • rmdir supprime un répertoire.
  • touch créé un fichier.
  • ll list all (ls-al) liste tout.
  • rm supprime.

Créer son projet en local

  • sudo mkdir -p /var/www/html
  • cd /var/www/html
  • sudo chmod -R 777 .
  • mkdir project_name
  • cd project_name
  • touch index.html
  • subl.

Versionner son projet sur GitHub

  • git init
  • git add index.html
  • git commit -m "initialize repository"
  • git push -u origin master

Gérer les conflits

Situation où 2 personnes modifient le même fichier en même temps

  • 1er Cas: modification sur GitHub

    • Dans GitHub → via l'onglet ‹ code ›
    - Modifier le code
    - Commit changes
    → update index.html (par exemple)
    Commit changes

  • 2e Cas: modification sur SublimeText (ou autre)

    • Pour corriger les conflits, on efface les caractères spéciaux accompagner de HEAD et master + les modifications qu'on ne veut pas.

    Après avoir corriger les conflits

    Dans le Terminal:

    • Merger une branche au Master après les modifications (on merge la branche depuis Master):
      • git checkout master
      • git merge branch_name
    • Une branche merger ne sert plus à rien. Il faut donc l'effacer:
      • git branch -d branch_name Demande si sur d'effacer? Non
      • git push origin master :branch_name
